Выполним всего две команды для настройки Git - ваше имя и почта

  1. git config --global user.name "Иван Иванов"
  2. git config --global user.email "ivanov@example.com"

Инициализировать (создать) репозитория в пустой папке

git init

Пример создания репозиторий в пустой папке

ФИКСАЦИЯ ИЗМЕНЕНИЙ

Посмотрим статус нашего репозитория

git status

Передаём файлы под контроль git

git add .

Передаём файлы под контроль git по имени файла

git add имя файла

Пример

git add file-respons.html

Если нужно добавить несколько файлов то файлы добовляются через пробел некакие запятые не ставятся ($ git add file-respons.html file-moon.php file-set.php и.т.д)

Фиксируем изменения (радактор Vim)

git commit

Мини шпаргалка по Vim

  1. Выход без сохранения изменений: — :q!
  2. Выход с сохранением изменений: — :wq
  3. Вставка текста там, где стоит курсор: — i
  4. Вставка текста в конце текущей строки: — A

Фиксируем изменения

git commit -m "текст коментария"

Используйте эту команду для коммитов если не хотите оказаться в редакторе Vim

Можно объединить git add и git commit в одну команду

git commit –a –m "текст коментария"

Удаляет файл из индекса и из рабочей папки (требует фиксации)

git rm FILE – имя файла

Пример

git rm FILE - file-respons.html

Удаляет файл только из индекса git (требует фиксации)

git rm --cached FILE – имя файла

Пример

git rm --cached FILE – file-respons.html

ИСТОРИЯ ИЗМЕНЕНИЙ

Просмотр лога коммитов

git log

Просмотр подробного лога коммитов

git log --p

Откат файла к последнему чистому состоянию (отмена индексации)

git checkuot -- имя файла

Пример

git checkuot -- file-respons.html

Откат всех файлов к последнему чистому состоянию

git checkuot --all

Откат к полследнему чистому состоянию репозитория (отмена индексации)

git reset HEAD

Откат к указанному коммиту с потерей всех изменений

git reset --hard HEAD

Ключ HEAD озночает к последнему комиту

Откат к указанному числом коммиту с потерей всех изменений

git reset --hard HEAD~число

Пример

git reset --hard HEAD~5

Также можно откатываться по хеш коду комита по полному коду или по первым 4м символам

  1. git reset --hard полный хеш код
  2. git reset --hard первые 4 символа
  1. git reset --hard ca82a6dff817ec66f44342007202690a93763949
  2. git reset --hard ca82

Добавление изменений к последнему коммиту

git commit --amend

Пример использования

  1. git commit -m "текст коммита"
  2. git add file-respons.html
  3. git commit --amend

УДАЛЕННЫЙ РЕПОЗИТОРИЙ

Добовляет к текущему ропозиторию ссылку на удалённый репозиторий

git remote add имя репозитория "адрес репозитория"

Пример

git remote add repositgit "https://github.com/repositgit/git.test.git"

Покозать список пикреплённых репозиториев

git remote

Покозать подробный список пикреплённых репозиториев

git remote -v

Взять информацию об изменениях в удалёном репозеторие. (Берётся только информация об изменениях сами изменения не перетаскиваются.) ps: Команда вытягивает например список веток или список комитов но не применяет их к нам

git fetch имя удалёного прикреплёного репозитория

Пример

git fetch repositgit

Взять подробную информацию об изменениях в удалёном репозеторие

git fetch show имя удалёного прикреплёного репозитория

Пример

git fetch show repositgit

Cвязать текущую локальную ветку с веткой удалёного репозитория

git checkout --track имя удалёного репозитория/master

Пример

git checkout --track repositgit/master

Кланируем удалёный репозиторий с GitHub

git clone (адрес репозитория с github) (имя папки в которую клонируем)

Пример по HTTPS

git clone https://github.com/repositgit/git.test.git my folder

Пример по SSH

git clone git@github.com:user/git-test.git my folder

Отправить сделанные изменения в удалённый репозиторий на GitHub

git push

Забрать изменения с удалёного репозитория если они есть. (Не просто «достаёт» изменения, но и пытается применить изменения с удаленного репозитория к нашему. (Влить удаленную ветку в локальную))

git pull

ПОНЯТИЕ «ВЕТКИ»

Создание ветки

git branch имя ветки

Пример

git branch additional-branch

Переключится на ветку

git checkout имя ветки

Пример

git checkout additional-branch

Создать новую ветку и сразу переключится на неё

git checkout -b имя ветки

Пример

git checkout -b additional-branch

Пасмотреть какие ветки есть

git branch -v

Удаление ненужных веток

git branch -d имя вети

Пример

branch -d additional-branch

Слияние веток

git merge имя ветки

Пример

git merge additional-branch

Для слияния веток нужно переключитя на ветку master и выполнить команду.
