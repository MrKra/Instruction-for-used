Регистрация на Github Github

Всего две настройки – ваше имя и почта:

$ git config --global user.name "Иван Иванов"

$ git config --global user.email ivanov@example.com

Создаем репозиторий в пустой папке:

$ git init

ФИКСАЦИЯ ИЗМЕНЕНИЙ

Посмотрим статус нашего репозитория:

$ git status

Передаём файлы под контроль git:

$ git add .

Передаём файлы под контроль git по имени файла:

$ git add имя файла

фиксация изменений:

$ git commit –m "комментарий"

Можно объединить git add и git commit в одну команду:

$ git commit –a –m "комментарий"

удаляет файл из индекса и из рабочей папки (требует фиксации)

$ git rm FILE – имя файла

удаляет файл только из индекса git (требует фиксации)

$ git rm --cached FILE – имя файла

ИСТОРИЯ ИЗМЕНЕНИЙ

просмотр лог коммитов

$ git log

просмотр подробного лога коммитов

$ git log --p

откат к полследнему чистому состоянию репозитория

$ git reset HEAD

откат файла к последнему чистому состоянию

$ git checkuot -- имя файла

откат всех файлов к последнему чистому состоянию

$ git checkuot --all

откат к указанному коммиту с потерей всех изменений

$ git reset --hard HEAD (ключ HEAD озночает к последнему комиту)

$ git reset --hard HEAD~3 (ключ HEAD~3 озночает откат на 3 комита назад)

Также можно откатываться по хеш коду комита по полному коду или по первы 4м символам

$ git reset --hard ca82a6dff817ec66f44342007202690a93763949

$ git reset --hard ca82

УДАЛЕННЫЙ РЕПОЗИТОРИЙ

Добовляет к текущему ропозиторию ссылку на удалённый репозиторий

$ git remote add имя репозитория "адрес репозитория"

Покозать список пикреплённых репозиториев

$git remote

Покозать подробный список пикреплённых репозиториев

$git remote -v

Взять информацию об изменениях в удалёном репозеторие (берётся только информация об изменениях сами изменения не перетаскиваются ps: например список веток или список комитов но не применяет их к нам )

$ git fetch имя удалёного прикреплёного репозитория

Взять подробную информацию об изменениях в удалёном репозеторие

$ git fetch show имя удалёного прикреплёного репозитория

Cвязать текущую локальную ветку с веткой удалёного репозитория

$ git checkout --track name repository/master

Кланируем удалёный репозиторий с Github

$ git clone адрес репозитория с github имя папки в которую скопировать

Пример: $ git clone https://github.com/repositgit/git.test.git) my folder

Отправить сделанные изменения в удалённый репозиторий на github

$ git push

Забрать изменения с удалёного репозитория если они есть (не просто «достаёт» изменения, но и пытается применить изменения с удаленного репозитория к нашему (влить удаленную ветку в локальную))

$ git pull

ПОНЯТИЕ «ВЕТКИ»

Создание ветки

$ git branch имя ветки

Пример:  $ git branch additional-branch

Переключится на другую ветку

$ git checkout имя ветки

Пример:  $ git checkout additional-branch

Создать новую ветку и сразу переключится на неё

$ git checkout -b имя ветки

Пример:  $ git checkout -b additional-branch

Пасмотреть какие ветки есть

$ git branch -v

Удаление ненужных веток

$ git branch -d имя вети

Пример:  $ branch -d additional-branch

Слияние веток (переключаемся на ветку master) и пишем коменду

$ git merge имя ветки

Пример:  $ git merge additional-branch
